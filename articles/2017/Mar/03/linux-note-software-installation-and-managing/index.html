Linux学习笔记 - Linux下软件的安装与管理

Posted by Ro$es in Technology

Tags Linux  CentOS 

参考资料：《循序渐进Linux第二版》

CentOS下软件安装方式主要有：

* 源码编译安装
* RPM安装
* yum安装
* 二进制安装

源码安装

步骤：

1. 下载解压源码
2. 分析安装平台环境（configure脚本）
3. 编译安装（make && make install）

注：在./configure时一定要注意使用--prefix参数指定安装目录（首选），或者在make && make install时注意保存日志，方便日后管理软件。（主要是删除软件时好找位置）

相关资料链接：Linux编译安装中configure、make和make install 命令详解 | 极安全-JiSec

RPM安装

RPM(Red Hat Package Manager)，Red Hat 软件包管理器，现已成为Linux平台下通用的软件包管理方式。 .rpm结尾的软件包为RPM文件，包含了已经编译好的二进制可执行代码。 源码进行编译、安装、封装，成为RPM文件。

优点：安装简单、方便。通过RPM安装的软件，会记录软件的安装信息，方便日后的查询、升级和卸载。 缺点：对操作系统环境的依赖很大。

RPM包封装格式有两种：RPM（.rpm）、SRPM（.src.rpm ）。

RPM安装时若出现依赖错误，要先安装依赖软件，然后继续安装需要的软件。

常用参数： 安装时：-ivh 安装，显示详细信息并使用"#"符号表现安装过程。 查询：-ql 安装后包名 显示软件包安装后的文件列表

yum安装

yum(yellowdog updater modified)，配合互联网即可实现软件的便捷安装和自动升级。 实质：指定远程的互联网主机上的RPM软件包，然后自动进行安装，同时解决各个软件之间的依赖关系。

二进制安装

二进制压缩包安装

一般为.tar.gz或.bz2后缀，直接解压可用。

有安装脚本的软件包

解压后，目录下一般有安装脚本，名称类似setup.sh或者install等，赋予可执行权限后运行即可完成安装。
