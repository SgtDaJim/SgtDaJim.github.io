Linux学习笔记 - Shell简介

Posted by Ro$es in Technology

Tags Linux  CentOS  shell 

参考资料：《循序渐进Linux第二版》

shell是围绕在Linux之外的一个“壳”程序，用户在操作系统上完成的所有任务都是通过shell和Linux系统内核的交互来实现的。
Linux下的shell有Bourne again shell (bash)[缺省] 、C shell (csh)、Korn shell (ksh)、Bourne shell (sh)和Tenex C shell (tcsh)等。

现在的Linux发行版一般都以bash作为默认shell。

shell本身是一个以C语言编写的程序，是用户和操作系统内核之间通信的桥梁。shell既是一种命令解释程序，又是一种功能强大的解释型程序设计语言。作为命令解释程序，shell解释用户输入的命令，然后提交到内核处理，最后把结果返回给用户。

为了加快命令的运行，同时更有效的定制shell程序，shell中定义了一些内置命令。
当用户登录系统后，shell以及内置命令就被系统载入到内存，并且一直运行，直到用户退出系统为止。
其他可执行文件与shell内置命令不同，当它们被调用时，才会被系统装入内存执行。

字符界面中，“#”表示登录的是系统超级用户，“*”表示登录到系统的是普通用户。

shell执行命令解释过程：

shell命令的语法分析

对于一些普通的命令，各个shell版本的语法基本相同，只有在编写一个shell脚本或者使用一些shell高级特性的时候，各个版本的shell的差异才会显示出来。

shell语法分析：shell对命令的扫描处理过程。
包括：重定向、文件名扩展和管道等。

以bash为例

命令行格式 

command [options] [arguments]
  • command 命令名称
  • options 命令选项
  • arguments 命令参数

在选项前一般有“-”符号，用于区别参数。

rm -rf /*

选项可以分开列，也可以单独列。

rm -r -f /*
或
rm -rf /*

有些命令不需要参数。若参数不够，则出现错误提示。

同一行可以多个命令，用“;”分开。

一个命令可以分行输入，用“\”分行。

通配符（可以组合使用）：

  • “*” 匹配任意一个或多个字符

  • “?” 匹配任意单一字符

  • “[]” 匹配方括号内所有的单个字符

shell的重定向

Linux下系统打开3个文件，即标准输入、标准输出和标准错误输出。 用户的shell将键盘设为默认的标准输入，默认的标准输出和标准错误输出为屏幕。

输入重定向

改变命令的输入源。利用输入重定向，可以将一个文件的内容作为命令的输入，而不从键盘输入。
操作符：'<' '<<'

输出重定向

将输出结果输出到一个指定文件中，而不是在屏幕。
操作符：'>' '>>'

错误重定向

与输出重定向类似。
操作符：'2>' '2>>'

shell的管道

管道可以把很多命令连接起来，可以把第一个命令的输入当作第二个命令的输出，第二个命令的输出当作第三个命令的输入，依此类推。
管道符：'|'

查看系统中正在运行的httpd进程。

ps -ef | grep httpd

shell中的引用

引用即为通知shell将一些有特殊含义的字符当作普通字符处理。
转义字符\，单引号''，双引号""

将文件“abc*?”重命名为“abc”

mv abc\?\* abc
或
mv 'abc?*' abc

双引号用法与单引号基本相同，但某些特殊字符在双引号中依然起作用。

从图得知，"$","\" ,"`" 在双引号中依然起作用。

使用tab键可以自动补全。
