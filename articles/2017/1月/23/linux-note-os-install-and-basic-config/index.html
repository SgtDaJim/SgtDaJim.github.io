Linux学习笔记 - Linux系统的安装与基础配置

Posted by Ro$es in Technology

Tags Linux  CentOS 

参考资料：《循序渐进Linux第二版》

安装方式

Linux的安装方式分为硬盘安装、U盘安装、网络安装和光驱安装。
可以使用Kickstart工具进行无人值守的Linux系统自动安装。

分区命名方案

/dev/hda2
/dev/sdb3
/dev ：所有设备文件的存放目录
hd和sd：分区的前两个字母，代表该分区所在的设备类型。hd代表IDE硬盘，sd代表SCSI硬盘。
a：分区命名的第三个字母，表示分区在哪个设备上。/dev/hda 代表第1块IDE硬盘，/dev/sdb代表第2块SCSI硬盘。
2：数字代表分区。Linux下前4个分区（主分区或扩展分区）用数字1~4表示，逻辑分区从5开始。/dev/sdb3代表第2块SCSI硬盘的第3个主分区或扩展分区。/dev/sdc6代表第3块SCSI硬盘的第2个逻辑分区。

Windows+Linux双系统注意事项

两种系统的文件系统不兼容。要从Windows系统完全删除分区，余出空间，从而安装Linux。
最好先安装Windows。因为每次安装Windows都会重新修改系统引导文件。安装Windows后Linux可能会没引导。

CentOS安装过程中的Troubleshooting模式

Troubleshooting 主要用于帮助解决各种安装问题以及对系统的故障修复。
Troubleshooting中的各个选项：

  • 图形模式下安装CentOS
  • 救援CentOS模式（最常用）（可用于系统内核问题、配置文件错误、磁盘错误等造成系统无法启动时的故障修复）
  • 内存测试
  • 从本地驱动器启动

Linux安装中的一些注意事项

对于Linux服务器来说，安全是第一位的，需要什么就安装什么，不需要的一定不要安装，多余的软件不但占用磁盘空间，而且会给服务器带来潜在的安全隐患。因此，当Linux作为一个服务器来使用时，要遵循按需安装、不用不装的原则。比如，要搭建一个DNS服务器，仅仅需要安装DNS软件包和一个基础系统内核即可。这样可以最大限度地提高系统效率和保障系统安全。

Linux系统下必须的分区为根分区（用“/”标识）和交换分区（swap）。
swap分区相当于Windows中虚拟内存的概念，也就是内存数据与硬盘的交换。
swap分区大小基本原则：如果内存较小（例如物理内存小于4GB），一般设置swap分区大小为物理内存的2倍；如果大于4GB而小于16GB，可以设置swap分区大小等于物理内存；如果内存大小在16GB以上，可以设置swap分区大小为0。但并不建议设置为0，一定大小的swap分区还是有作用的。

只划分根分区和swap分区可以完成安装但不建议。一旦根分区遭到破坏，系统无法启动，根分区中资料也可能丢失。建议给独立的应用分配独立的分区，能最大限度减少系统崩溃造成的损失。

建议独立分配的分区：

  • /boot：存储系统的引导信息和内核等信息
  • /usr：存储系统应用软件安装信息
  • /var：存储系统日志信息
